# Workflow

## Purpose

Provide a consistent delivery flow from plan to implementation and verification.

## Inputs

- task goal
- repository context
- available quality gates

## Process

1. Clarify outcome and constraints.
2. Implement in small, testable steps.
3. Run quality gates relevant to the repo.
4. Summarize decisions and remaining risks.

## Output Format

- Goal
- Changes made
- Verification performed
- Remaining risks

## Quality Checks

- [ ] No unrelated refactors
- [ ] Verification commands executed
- [ ] Risks documented when checks cannot run
